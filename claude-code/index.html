一、claude 安装

1、安装 Git Bash（仅 Windows 需要，Mac / Linux 跳过）

Claude Code 原生是为 Linux / macOS 设计的，而 Windows 系统命令不同，所以需要装 Git 来进行一个准备工作。

下载安装包，安装时狂点下一步就行。

Git Bash下载地址：https://git-scm.com/install/windows

2、安装 Claude Code 本体

打开终端（Windows 用 PowerShell，Mac/Linux 用 Terminal），粘贴命令即可：

Windows

irm https://daheiai.com/cc.ps1 | iex

Mac / Linux

irm https://daheiai.com/cc.ps1 | iex

安装 2.1.153 老版本命令：

& ([scriptblock]::Create((irm https://daheiai.com/cc.ps1))) 2.1.153

备用：官方原版命令 irm https://claude.ai/install.ps1 | iex

因为 2.1.156 版本使用第三方 API 存在兼容性问题，建议回退 153 版本。

整个过程会下载一个 200+MB 的软件，取决于个人的网络状态，时间可能会很长。

3、配置环境变量 PATH

把 C:\Users\你的用户名\.local\bin\ 加到用户级 PATH。

Windows

具体操作

点开开始菜单，输入环境变量，点“编辑系统环境变量”。

在窗口下面点“环境变量”，在系统环境变量里找到 Path，双击它。

新建一条，把 C:\Users\你的用户名\.local\bin\ 复制进去。

Mac / Linux

通常不需要手动配置

Mac / Linux 的安装脚本一般会自动配置 PATH。

先试试关掉当前终端，重开一个新的输入 claude。

然后还是找不到命令，手动把以下内容加到你的 shell 配置文件中：

export PATH="$HOME/.local/bin:$PATH"

加到 ~/.bashrc 或 ~/.zshrc，然后执行 source ~/.zshrc 生效。

4、安装 cc-switch，接入更多模型

软件装好了，但还没有 AI 模型可用。这时候需要 cc-switch，它可以切换不同的 AI 模型提供商。

cc-switch 支持的模型

智谱 GLM、MiniMax、Kimi、DeepSeek等

Claude、GPT 等模型可通过 API 服务 接入。

下载安装：https://github.com/farion1231/cc-switch/releases

安装好了之后打开cc-Switch，按照如下图添加AI大模型

以智谱GLM为例，先去智谱 GLM官网注册账号获取API key,官网地址：https://www.bigmodel.cn/glm-coding?ic=A7DYW5XKAV

获取API key之后，先复制API key，然后再回到cc-Switch，点击右上角“+”号添加大模型，选择Zhipu GLM，输入API key，点击右下角“添加“按钮，就添加成功了。

接着再启动该模型，咱们的claude就可以使用智谱 GLM模型了。

二、Cursor IDE使用claude

1、Cursor官网下载：https://cursor.com/cn

打开Cursor，打开命令终端（快捷键Ctrl+J）,输入命令：claude，启动claude。

2、claude code的三种权限模式

此外claude code(后面简称CC)还有一种风险比较高的模式，启动方式，在命令终端输入： claude --dangerously-skip-permissions

顾名思义就是有一点风险，执行任务的时候不需要用户任何确认，包括修改、删除文件等，我个人用的是默认模式。

4、CC高级扩展

1、Skill

不是每次都需要使用，不需要写到claude.md文件中，但是这些功能又很常用，所以我们可以把它封装成skill，等需要使用的时候再调用它，Skill一般分为以下4种类型：

那如何找到这些Skill呢？

我们可以先下载一个Find Skill，它的功能就是帮我找到我们需要使用的Skill，把Find Skill的下载链接告诉claude，让它自动帮我们下载和安装就可以了，如下图：

Find Skill下载链接：https://github.com/vercel-labs/skills

下载之后会提示：

● 安装完成！find-skills 技能已就绪：

  - 位置：.claude/skills/find-skills/
  - 功能：当你说"帮我找一个能做 X 的技能"时，该技能会自动搜索 skills.sh 上的技能并推荐安装
  - 使用方式：直接问我"有没有可以 XX 的技能"即可触发，

如果我们再想要什么功能或者什么类型的Skill，就可以直接和claude说，它就一键帮我们搞定了，比如可以直接和claude说：我想要一个优化前端的skill，它就会自己帮我们安装这个skill，自己可以手动选择安装在全局和当前项目下。

2、openCLI

openCLI把我们常用的应用以及社交媒体封装成了skills，安装openCLI之后，可以直接通过claude访问软件应用以及社交媒体，把openCLI的链接告诉claude，它就可以自动帮我们安装了；

openCLI下载链接：https://github.com/jackwener/OpenCLI/

可以看到它真的就从小红书上给我们汇总了10家必吃的餐厅，当然你的提示词还可以跟详细一点，比如按照距离远近推荐、对应的餐厅图片也反馈给我等。

还可以让claude帮你去浏览知乎、B站等其他媒体软件，彻底解放你的双手。

3、子agent

有时claude在写代码或者执行任务的时候需要去调研或者其他分支任务，可以让claude主动分配几个子agent分别去干某些任务，这样不影响任务。

5、Hooks

Hooks 是 Claude Code 的事件驱动自动化系统：在特定事件触发时，自动执行一组 Hook。

它主要有两种形态：

命令型 Hooks（command）：运行 Shell 命令/脚本，适合自动化与校验（格式化、检查、通知、跑测试……）。
提示型 Hooks（prompt）：在 Stop / SubagentStop 事件里，让 Claude 再做一次“停下前的质量检查”，根据结果决定继续还是停止。
你可以把它理解为：

SessionStart/End：会话生命周期钩子
PreToolUse/PostToolUse：工具执行前/后钩子（Write/Edit/Bash/Read…）
Stop/SubagentStop：模型准备结束前的“收尾门禁”
Notification：系统通知的自定义处理

比如可以跟claude说：帮我做个Hook，每次任务完成之后，自动发送一个提示音。

它就会在每次任务完成后自动提醒我们。
