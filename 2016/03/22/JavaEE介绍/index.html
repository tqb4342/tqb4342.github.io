JavaEE介绍

JavaEE

  • HTTP:Tomcat
  • Servlet JSP JSTL EL
  • MVC设计模式
  • xml
  • AJAX:JS
  • JQuery：js框架

    软件的架构

    C/S:Client/Server(胖客户端)

    优点：可以发挥客户端计算机的硬件，功能更强大，缓存；
    缺点：维护性差

    B/S：Browser/Server(瘦客户端)

    优点：维护性特别好，
    缺点：服务器压力比较大

    RIA：rich internet application(富客户端)

    ajax jquery extjs javaFx Flex html5

    HTTP协议：

  • hyper text transfer protocal ：80 超文本传输协议，是TCP协议的高层协议
  • 特点：无状态 无连接 基于请求/响应模式的协议
  • URL：统一资源定位器

    格式：协议://ip地址:端口号/应用名/资源？参数(http://www.taobao.com )

    HTTP1.0与HTTP1.1的区别

  • 如果在一个页面中包含其它资源（图片，js，css等），http1.0会创建多个连接，每个连接发送一个请求，而http1.1只会创建一个连接，在一个连接中发送多个请求，性能更好。

    HTTP协议的结构：

    三部分：

            1.请求行
         2.消息头
         3.内容体（可选）
    响应的三部分
    1.状态行
    2.消息行
    3.实体内容

    状态行中有状态码，状态码常用的数值是

             200：服务器正常处理
         404：客户端错误，找不到页面
         500：服务器内容错误，通常是程序错误

GET与POST的区别

get通过请求行发送数据
当直接在地址栏上写URL 使用A标签 使用js的location，open 或者form表单中method的属性值位get时，发送时get请求
post通过请求体发送数据
post更好，对长度没有限制
form表单中method的属性值位post时，发送post请求，尽量使用post请求
