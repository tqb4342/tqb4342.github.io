一、注册账号与实名登记

首先我们需要在官网里面注册一个账号，并且完成实名认证，一般是注册【个人】的身份。 中国版权保护中心官网

中国版权保护中心www.ccopyright.com.cn/

注册完成后点击下面进行登记，如果没有实名需要先实名认证（认证时间一般在3个工作日）

实名认证需要

1、身份证正面

2、身份证反面

3、手持身份证照片

二、材料准备

在认证审核的过程中我们就可以着手开始准备材料了。如果是多人合作开发著作权人不是一个人的话，还需要准备一份合作开发协议

  • 程序源代码，需要的是pdf类型的文件
  • 用户手册，需要pdf类型的
  • 合作开发协议书

以下是一些模板示例文件提供给大家参考：

百度网盘链接：

  • 链接: https://pan.baidu.com/s/15kwMEiKZ5G_khnJBUMxouA?pwd=picz
  • 提取码: picz

源代码：源程序前连续的30页和后连续的30页

用户使用手册：提交登记软件的任何一种文档的连续的前30页和连续的后30页，每页不少于30行。若整个文档不到60页，应提交整个文档。可以是用户使用手册、程序说明书等

合作开发协议书：这个如果是著作权人有多个（多个人一起开发）就需要提供这个材料

三、申请步骤

第一步，我们先选择计算机软件著作权登记申请，第一个图

1.办理身份

这里我们选择我是申请人

2.软件申请信息

  • 权利取得方式：原始取得。
  • 软件全称：XX系统。比如我的是 XXXX博客系统。软件全称应简短明确，申请材料中的软件全称应保持一致
  • 软件简称：如果没有就空着 ，如果有，不要与软件全称完全相同。
  • 版本号：V1.0。建议按规范填写，如V1.0或1.0这两种形式。
  • 权利范围：全部权利。

3.软件开发信息

按照实际情况填写就可以了。

  • 软件分类：应用软件。前后端系统一般是应用软件。

  • 软件说明：原创。

  • 开发方式：单独开发。（其他方式需要提交相应的协议书）

  • 完成开发日期：选择完成开发的时间即可。

  • 发表状态：未发表。（如果是已发表，需要填写首次发表日期和首次发表地点）

  • 著作权人：

  • 国家：中国

  • 地区：学校所在省市区

  • 身份类别：事业单位法人 名称：学校名称

  • 统一社会信用代码证书：网上一般可以找到

  • 注意：著作权人如果是合作开发可以写多个，但是需要其他著作权人也注册该网站，且完成实名认证。后面需要获取每个人的手机号验证码才能继续。

4.软件功能与特点

这一页的内容比较多，可以参考我下面的

  • 开发的硬件环境：四核intel i5 CPU，16GB内存，512固态硬盘，10Mbps网络带宽
  • 运行的硬件环境：服务端内存2G以上，硬盘空间不低于40G； 客户端4G及以上内存，硬盘空间16G及以上
  • 开发该软件的操作系统：Windows 10操作系统
  • 软件开发环境 / 开发工具：开发环境：JDK8，node.js16.17.1，Vue3 开发工具：IntelliJ IDEA VsCode
  • 该软件的运行平台 / 操作系统：Cent OS 8.9
  • 软件运行支撑环境 / 支持软件：JDK8，nginx-1.24.5，MySQL5.0及以上版本
  • 编程语言：Java JavaScript HTML SQL Python
  • 源程序量：实际的代码行数 Java代码行数：IDEA插件Statistic可以统计行数 vue代码行数统计方法：在项目根目录打开git bash,执行下列代码 find ./src “(“ -name “.html” -or -name “.vue” -or -name “.js” -or -name “.ts” -or -name “.css” -or -name “.styl” -or -name “.less” -or -name “.scss” “)” -print | xargs wc -l
  • 开发目的（可以借助AI工具来写）
  • 面向领域 / 行业：教育行业等…
  • 软件的主要功能（可以借助AI工具来写
  • 软件的技术特点：本系统具有简单性、面向对象、健壮性、平台独立与可移植性等特点。
  • 程序鉴别材料：提交登记软件源程序连续的前30页和连续的后30页（前、后各连续30页可以按开发时间排序，也可以按功能主次等自定义排序），每页不少于50行。若源程序整体不到60页，应提交全部源程序。
  • 文档鉴别材料：提交登记软件的任何一种文档的连续的前30页和连续的后30页，每页不少于30行。若整个文档不到60页，应提交整个文档。文档是指用来描述程序的内容、组成、设计、功能规格、开发情况、测试结果及使用方法的文字资料和图表等，如程序设计说明书、流程图、用户手册等。
